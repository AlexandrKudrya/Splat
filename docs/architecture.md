# Архитектура SplitCheck

## Обзор

SplitCheck — Telegram Mini App для разделения счёта. Пользователь фотографирует чек, Claude API распознаёт позиции, каждый участник отмечает что он ел, и приложение считает кто сколько должен.

## Стек

| Слой | Технология |
|---|---|
| Backend | Python 3.12, FastAPI, SQLAlchemy 2 (async), Alembic |
| Bot | Python 3.12, aiogram 3 |
| База данных | PostgreSQL 16 |
| AI | Anthropic Claude API (`claude-opus-4-6`) |
| Хранилище фото | S3-совместимое (AWS S3 / Minio) |
| Авторизация | Telegram initData → JWT (HS256) |
| Деплой | Docker Compose |

## Жизненный цикл заказа

```
                     ┌─────────┐
                     │  DRAFT  │  ← создан, фото не загружено
                     └────┬────┘
                          │ POST /orders/{id}/photo
                          ▼
                     ┌─────────┐
              ┌──────│ PENDING │  ← фото загружено, Claude парсит чек
              │      └────┬────┘
              │ ошибка    │ Claude успешно распознал позиции
              ▼           ▼
         ┌────────┐  ┌─────────┐
         │ FAILED │  │ ACTIVE  │  ← все участники выбрали свои позиции
         └────────┘  └────┬────┘
                          │ все участники отметили оплату
                          ▼
                     ┌──────────┐
                     │ FINISHED │
                     └──────────┘
```

### Переходы

| Откуда | Куда | Триггер |
|---|---|---|
| `DRAFT` | `PENDING` | Claude успешно распознал чек (фоновая задача) |
| `DRAFT` | `FAILED` | Claude вернул ошибку при парсинге |
| `PENDING` | `ACTIVE` | Последний участник выбрал свои позиции |
| `ACTIVE` | `FINISHED` | Последний участник отметил оплату |

## Авторизация

1. Telegram Mini App получает `initData` от Telegram
2. Фронтенд отправляет `initData` на `POST /api/v1/auth/telegram`
3. Backend проверяет HMAC-SHA256 подпись с `TELEGRAM_BOT_TOKEN`
4. Возвращает JWT-токен (срок жизни: `ACCESS_TOKEN_EXPIRE_MINUTES`, по умолчанию 24 часа)
5. Все последующие запросы идут с заголовком `Authorization: Bearer <token>`

## Парсинг чека (фоновая задача)

1. `POST /orders/{id}/photo` — загружает фото в S3, сохраняет URL в `orders.photo_url`
2. FastAPI `BackgroundTasks` запускает `_parse_receipt_background`
3. Задача скачивает фото по URL и отправляет в Claude API с промптом
4. Claude возвращает JSON с позициями чека
5. Позиции сохраняются в `orders.order_info` (JSONB), статус меняется на `PENDING`
6. При ошибке статус меняется на `FAILED`
